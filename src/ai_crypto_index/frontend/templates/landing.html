How it works

How each monthly run becomes portfolio-ready

From market data to published weights, every step is designed to deliver stable diversification, measurable risk control, and immediate API-ready outputs.

  1. 01

    Capture market state

    Liquid-market quotes are consolidated into a clean baseline so every rebalance starts from consistent data.

  2. 02

    Turn data into signals

    Feature engineering translates price behavior into momentum and volatility signals that drive allocation decisions.

  3. 03

    Estimate forward risk

    LSTM volatility forecasts and covariance estimates model near-term risk before weights are finalized.

  4. 04

    Build diversified weights

    A risk-parity optimizer applies allocation caps and liquidity constraints to keep concentration under control.

  5. 05

    Publish and monitor outcomes

    Each monthly run ships with run IDs, performance history, and downloadable artifacts in dashboard and API.

A Run ID identifies one specific pipeline execution (scheduled rebalance or on-demand run). It’s the “receipt” that ties together weights, performance metrics, and exported files for that exact run.
