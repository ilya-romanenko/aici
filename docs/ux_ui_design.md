# Этап 3. UX/UI дизайн

## Резюме этапа
- Собраны адаптивные вайрфреймы главного лендинга (desktop/tablet/mobile), учитывающие последовательность блоков из `docs/roadmap.md`.
- Обновлён UI-кит: цветовые токены, типографика, сетка, карточки, таблицы, элементы форм и состояния.
- Проработаны финальные макеты для трёх брейкпоинтов с учётом интерактивных элементов и контента из `docs/web_copy_guidelines.md`.
- Утверждены сценарии микроанимаций и поведение CTA-кнопок для поддержания технологичного и доверительного тона.

## Вайрфреймы главной страницы

**Глобальные принципы**
- Базовая сетка: desktop — 12 колонок (ширина колонки 72px, промежуток 24px), tablet — 8 колонок (60/16), mobile — 4 колонки (66/16).
- Основной контейнер: desktop 1200px, tablet 88% ширины экрана, mobile 92%.
- Вертикальные отступы между блоками: 120px (desktop), 96px (tablet), 72px (mobile). Внутренние отступы секций — 48/32/24px.
- Глобальная шапка фиксируется после прокрутки 120px, высота 72px (desktop), 64px (tablet), 56px (mobile).

### Дополнительные артефакты
- Состояния `hover` и `focus` прорисованы в вайрфреймах для CTA и ссылок.
- Для бургер-меню предусмотрены отдельно каркасы открытого состояния (tablet/mobile).
- Варианты темной и светлой версии отмечены слоями `Theme/Dark` и `Theme/Light`.

## UI-кит

### Цветовые токены
| Токен | HEX | Назначение |
| --- | --- | --- |
| `color-primary` | #1A237E | Акцент, CTA, важные ссылки |
| `color-primary-contrast` | #FFFFFF | Текст на первичном фоне |
| `color-secondary` | #00E5FF | Подсветка графиков, градиенты, флоат-элементы |
| `color-bg-dark` | #121212 | Тёмный фон секций |
| `color-bg-light` | #F5F7FA | Светлый фон секций |
| `color-text-main` | #0D0D0D / #FFFFFF | Основной текст (светлая/тёмная темы) |
| `color-text-secondary` | #9E9E9E | Вторичные подписи, лейблы |
| `color-success` | #4CAF50 | Положительная динамика |
| `color-danger` | #FF5252 | Предупреждения, отрицательные значения |
| `color-border` | rgba(255,255,255,0.12) / rgba(18,18,18,0.12) | Разделители, бордеры |
| `color-surface-elevated` | rgba(26,35,126,0.12) | Ховер фон карточек |

### Типографика
- Базовый шрифт: `Inter` (fallback `Roboto`, `Arial`, `sans-serif`).
- Заголовки:
  - `H1`: 56/64px, `Inter SemiBold`, letter-spacing -0.5px.
  - `H2`: 40/48px, `Inter SemiBold`.
  - `H3`: 28/36px, `Inter Medium`.
- Подзаголовки и body:
  - `Body Large`: 18/28px, `Inter Regular`.
  - `Body`: 16/26px, `Inter Regular`.
  - `Caption`: 14/22px, `Inter Regular`, uppercase для лейблов.
- Числовые значения KPI: `Inter Mono` 24/32px для точной табличной выравнивающей сетки.

### Сетка и отступы
- Горизонтальный ритм: шаг 8px, ключевые значения 8, 16, 24, 32, 48, 64.
- Радиусы: 12px (карточки), 8px (кнопки), 6px (inputs), 999px (бейджи).
- Тени: `shadow-1` (0 16px 40px rgba(26,35,126,0.2)), `shadow-2` (0 6px 18px rgba(0,0,0,0.16)).

### Компоненты
- **Buttons**: Primary (`color-primary` фон, `color-primary-contrast` текст), Secondary (прозрачный фон, бордер `color-primary`), Tertiary (иконка + текст). Состояния:
  - `hover`: lighten на 8% + тень `shadow-2`.
  - `active`: смещение 2px вниз, тень убирается.
  - `focus`: внешний обвод 2px `color-secondary`.
- **Inputs**: Высота 52px, бордер 1px (`color-border`), фокус — подсветка `color-secondary`.
- **Cards**: Padding 32px, header с иконкой 48px, при hover - фон `color-surface-elevated`.
- **Table**: Заголовки 14px uppercase, зебра через `rgba(26,35,126,0.04)`. Sticky header на desktop.
- **Badges**: Размеры 28px высота, primary (#1A237E), success (#4CAF50), neutral (#9E9E9E).

### Памятка по стилям форм (классы вместо type)
- Общие контейнеры: `.landing-intake__field`, `.landing-intake__checkbox`, `.support-form__field`, `.support-form__upload`. Для частных кейсов добавляйте модификаторы на контейнер (`--email`, `--password`, `--textarea`, `--checkbox`).
- Элементы и состояния вешаются на классы (`__input`, `__textarea`, `__checkbox-input`, `.is-invalid`, `.is-disabled`, `.is-focused`); атрибутные селекторы `[type=email]`, `[type=password]`, `[type=checkbox]` не используем вне `normalize.css`.
- Примеры селекторов на контейнеры:
  - `.landing-intake__field--email .landing-intake__input { … }`
  - `.support-form__field.is-invalid .support-form__input { … }`
  - `.landing-intake__checkbox .landing-intake__checkbox-input { … }`
  - `.support-form__field--textarea .support-form__textarea { … }`

### Иконография и иллюстрации
- Линейные иконки 24/32px (stroke 1.5px) с плавными скруглениями.
- Иллюстрации (геро-сцена, графики) содержат сочетание #1A237E + #00E5FF градиентов, допускается добавление акцентного свечения `rgba(0,229,255,0.35)`.

### Токены в коде
```
:root {
  --color-primary: #1A237E;
  --color-secondary: #00E5FF;
  --color-bg-dark: #121212;
  --color-bg-light: #F5F7FA;
  --color-text-main: #0D0D0D;
  --color-text-main-dark: #FFFFFF;
  --color-text-secondary: #9E9E9E;
  --color-success: #4CAF50;
  --color-danger: #FF5252;
  --radius-card: 12px;
  --radius-button: 8px;
}
```

## Финальные макеты

### Desktop (1440px)
- Hero содержит стек из текстового блока и 3D-плитки «Realtime weights» с живыми цифрами, фон — градиент `#0D0D0D → #1A237E`, неоновая сетка в углу.
- Таймлайн «Как это работает» реализован как диаграмма с прогрессом, активный шаг увеличивается до 110%.
- Таблица состава индекса имеет закреплённый столбец с иконкой актива и кнопкой «Подробнее».

### Tablet (1024px)
- Header трансформируется в бургер-меню, CTA остаётся доступной кнопкой в правом углу off-canvas.
- Карточки ценности компонуются по две с адаптивными высотами, тексты центрируются.
- Блок тарифов — вертикальный список с раскрытием деталей внутри карточки.

### Mobile (390px)
- Hero размещает CTA одну под другой с верхним `Get demo access` и вторичным `View performance`.
- Все табличные данные переводятся в карточки с ключевыми метриками и иконками.
- FAQ раскрывает ответы поверх, автоматически прокручивая к верхней части карточки.

### Темные/светлые версии
- Для каждой секции подготовлены два слоя (`/light` и `/dark`) с согласованными контрастами WCAG AA (проверено для текста 16px и кнопок).
- CTA на темном фоне — светлый `color-secondary`, на светлом — первичный `color-primary`.

## Микроанимации и поведение CTA

| Элемент | Сценарий | Время / Кривая |
| --- | --- | --- |
| Primary CTA | Ховер — лёгкий glow `0 0 24px rgba(0,229,255,0.45)` + подъем на 2px; клик — обратный переход | 160ms, `cubic-bezier(0.4, 0, 0.2, 1)` |
| Secondary CTA | Подчеркивание «скользит» слева направо | 140ms, linear |
| Таймлайн шаг | При наведении активный шаг масштабируется 1.08, линия прогресса заполняется | 220ms, `ease-out` |
| Карточка ценности | Появление при скролле (fade + translateY 16px) | 260ms, `cubic-bezier(0.33, 1, 0.68, 1)` |
| График доходности | Анимация линии с `stroke-dashoffset` при загрузке, обновление данных с ease-in-out | 480ms, `ease-in-out` |
| Таблица индекса | Ховер строки — подсветка `color-surface-elevated`, кнопка экспорта появляется с opacity | 180ms, `ease-out` |
| FAQ | Раскрытие — height auto через `max-height` трюк, стрелка вращается на 90° | 200ms, `ease` |
| Бургер-меню | Иконка трансформируется в крест, меню выезжает справа с размытой подложкой | 320ms, `cubic-bezier(0.22, 1, 0.36, 1)` |

Дополнительно:
- Все анимации отключаются при `prefers-reduced-motion`.
- Ховеры сохраняют контраст не ниже 4.5:1.
- CTA записаны в прототипе с `Interaction → Open overlay`/`Navigate` для готовой передачи разработчикам.

## Рекомендации по передаче в разработку
- Экспорт ассетов: иконки в SVG, фоновые сетки в WebP 2x/1x.
- Стили и токены синхронизированы в storybook: `tokens.json` с вышеуказанными значениями.
- Проверочный чек-лист QA включает сравнение с вайрфреймами по keylines и контрольным отступам.
