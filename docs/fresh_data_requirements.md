# Требования: скрытая загрузка fresh data и кеш по n_top_coins

## Цель
Скрыть ручной параметр "Force fresh data" и перейти на автоматический выбор источника данных, чтобы базовые пользователи работали на предзагруженном дневном наборе n_top_coins=100 без лишних обращений за свежими данными.

## Термины
- **Базовый набор** — дневной снапшот Universe size n_top_coins=100, загружается один раз в начале дня.
- **Кастомный набор** — данные, полученные при запросе с n_top_coins != 100.
- **Fresh data** — принудительная загрузка актуальных данных для нужного n_top_coins.

## Требования к процессу загрузки
- В начале каждого календарного дня (UTC) автоматически загружать и сохранять базовый набор n_top_coins=100 без участия пользователя.
- Базовый набор использовать как источник по умолчанию для всех запросов с n_top_coins=100.
- Для кастомных значений n_top_coins загружать fresh data только при первом запросе на конкретное значение.
- После первой загрузки кастомного значения повторно использовать кеш вместо повторных fresh data.

## Ограничения по хранению
- Базовый набор хранить с дневной ротацией: один актуальный снапшот в день, устаревшие версии удалять или перезаписывать.
- Для кастомных наборов хранить не более 5-10 вариантов n_top_coins одновременно; при превышении лимита удалять самый старый.
- Кастомные наборы очищать пакетно раз в месяц (удаление всего набора или по сроку давности — уточнить при реализации).

## Правила доступа и тарифы
- Параметр "Force fresh data" скрыть на фронтенде (app/playground) для всех тарифов.
- В API/бэкенде запретить принудительное указание fresh data; выбор источника (кеш или загрузка) должен происходить автоматически.

## Логика выбора источника данных
- Если n_top_coins=100 — брать текущий базовый набор.
- Если n_top_coins != 100:
  - проверить наличие кастомного кеша;
  - если есть — вернуть из кеша;
  - если нет — загрузить fresh data, сохранить в кеш, вернуть пользователю.

## Мониторинг и валидация
- Фиксировать timestamp загрузки для каждого набора (базового и кастомных) и источник (fresh/cache).
- Логировать события вытеснения кеша по n_top_coins для аудита.
- Добавить алерт при отсутствии базового набора на текущий день или при ошибке его загрузки.

## Принятые решения по ранее открытым вопросам
- Время автозагрузки базового набора: 06:00 UTC (после закрытия дневных свечей); ретраи — 2 попытки с интервалами 10 и 20 минут. При окончательном фейле используем вчерашний снапшот с меткой «stale» и поднимаем алерт.
- Хранилище: S3-совместимый бакет `s3://ai-ci/daily/{yyyy-mm-dd}/n100.parquet` и `.../custom/{n}/snapshot.parquet`; доступ по IAM-ролям только для сервисов pipeline (ETL + API).
- TTL кастомных наборов: 7 дней без обращений; при каждом чтении обновляем `last_access`; чистка по `last_access`.
- Политика обновления кастомного кеша: всегда отдаём кеш, пока не истёк TTL; перезаписываем только после истечения TTL.
